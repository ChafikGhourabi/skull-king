<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GuestUpgradeModal → Supabase Auth | linkIdentity and updateUser calls; requires valid anon session |
| localStorage → component state | sk_upgrade_dismissed flag read on mount; writeable by any JS on the page |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-02 | Information Disclosure | Anonymous session on shared device | accept | Anon session is low-value (no PII until upgrade); GuestBadge makes guest status visible; sign-out is always available; documented in SKELETON.md |
| T-1-05 | Tampering | linkIdentity without Manual Linking enabled | mitigate | Code comment explicitly notes dashboard requirement; updateUser email path works without it as a fallback; toast.error surfaces any linkIdentity failure to the user |
| T-1-06 | Information Disclosure | localStorage sk_upgrade_dismissed flag | accept | Low-value flag; not a secret; cannot escalate privileges; XSS can write localStorage but writing this flag only suppresses a UI prompt |
| T-AUTH-03 | Repudiation | Game history on anonymous account | mitigate | updateUser and linkIdentity both preserve user_id — D-18 atomicity guaranteed by Supabase identity system; no data migration needed |
</threat_model>
