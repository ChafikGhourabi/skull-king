# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through actual failure this session.

- [ ] CONSTRAINT: Tailwind v4 spacing token naming — `--spacing-{name}` tokens in `@theme` silently override the Tailwind scale, breaking `max-w-sm`, `max-w-md`, etc. **ALREADY FIXED** (renamed to `--gap-*`) but must not re-introduce `--spacing-*` named tokens in theme.css.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `--spacing-*` token naming | Defining `--spacing-sm`, `--spacing-md` etc. in Tailwind v4 `@theme` overrides the spacing scale; `max-w-sm` became 8px. Fixed by renaming to `--gap-*`. | advisory | Always use `--gap-*` or `--size-*` for custom spacing aliases; never `--spacing-{name}`. |
| Migration forward-reference | SQL RLS policy on `games` referenced `game_players` before that table was created. Migration failed mid-apply. Fixed by creating all tables first, then all policies. | advisory | Always create ALL tables in a migration before any cross-table USING clauses in policies. |
| redirectIfAuthed blocks guests | `redirectIfAuthed` was redirecting ALL sessions (incl. anonymous) away from auth screens, stranding guest users on /home with no path to sign in. | advisory | Always check `session.user.is_anonymous` — anonymous sessions must reach /auth/* to upgrade. |
