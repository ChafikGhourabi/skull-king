    Create netlify.toml at project root — exact content per PATTERNS.md:
    ```toml
    [build]
      command   = "pnpm build"
      publish   = "dist"

    [[redirects]]
      from   = "/*"
      to     = "/index.html"
      status = 200
    ```
    The `[[redirects]]` rule is the critical SPA fallback — without it, direct navigation to /home or /auth/login returns a Netlify 404 instead of serving index.html. No additional config needed for scheduled functions — Netlify detects them from the `config` export.

    Create netlify/functions/keep-alive.ts per PATTERNS.md Pattern 6:
    - Import `Config` type from `"@netlify/functions"`
    - Default export: async arrow function `async () => { ... }`
    - Inside function: construct URL as `${process.env.VITE_SUPABASE_URL}/rest/v1/`; make a `fetch(url, { headers: { apikey: process.env.VITE_SUPABASE_ANON_KEY ?? '' } })` GET request
    - Add `console.log('[keep-alive] Supabase ping sent at', new Date().toISOString())` for Netlify function logs
    - Named export: `export const config: Config = { schedule: "0 0 */3 * *" }` — every 3 days at midnight UTC (Claude's discretion per D-15)
    - Note in file comment: "Prevents Supabase free-tier project from pausing after 7 days of inactivity (D-15)"

| keep-alive function → Supabase REST | Uses apikey header with anon key; read-only health check |
| GitHub push → Netlify CI | Build trigger on main branch push; no staging environment (D-13) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-SC | Tampering | npm/pnpm install during Netlify CI build | mitigate | pnpm-lock.yaml committed to repo — Netlify uses exact locked versions; Package Legitimacy Audit in RESEARCH.md confirms all packages |
| T-DEPLOY-01 | Information Disclosure | VITE_SUPABASE_ANON_KEY in built JS bundle | accept | Supabase anon key is intentionally public-safe (JWT that grants only anon role access); RLS prevents data access; documented Supabase security model |
| T-DEPLOY-02 | Information Disclosure | .env.local secrets committed accidentally | mitigate | .env.example (not .env.local) committed; .gitignore includes .env.local; Task 1 verifies this |
| T-DEPLOY-03 | Denial of Service | Supabase free-tier pause | mitigate | keep-alive function pings every 3 days — within the 7-day inactivity threshold; cron: "0 0 */3 * *" |
</threat_model>
